## Problem
1) 대규모 요청 환경에서 클라이언트가 대기열 상태 정보를 알지 못한채 대기하게 되어,
   connection timeout이 발생하며 요청을 성공적으로 수행하지 못함.
2) 대기 상태 정보를 알지 못해, 클라이언트는 요청 처리에 대해 예측하지 못함.

## Solution
1) queue에 적재된 요청을 요청의 unique key를 이용하여 map에 동시 저장
2) 저장된 map 상태를 주기적으로 snapshot 하여, 조회 요청이 map에 영향을 주지 않도록 설계
3) map을 저장/관리하는 스레드를 단일 스레드로 관리하여 동시성을 위한 불필요한 lock을 사용하지 않도록 설계

## Result
1) 조회 요청이 많아도 기존 요청을 처리하는 queue 처리 성능에 영향이 없도록 구현
2) 요청별 unique key를 이용하여 요청 상태를 조회하여 대기중인 요청의 근 실시간 상태를 조회
